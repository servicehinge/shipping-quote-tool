# 程式小白使用 Claude Code 的注意事項

> 這份文件是寫給「完全不懂程式，但想用 AI 來幫忙寫程式」的人。
> 你不需要變成工程師，但你需要知道哪些事情很重要、不能忽略。

---

## 一、安全性（最重要，沒有之一）

用 AI 寫程式的時候，最容易出事的就是**把密碼和金鑰搞丟了**。

### 什麼是「密鑰」？

你的程式可能需要連接到其他服務（例如付款系統、資料庫、第三方 API）。
這些服務會給你一組「鑰匙」（API Key、密碼、Token），讓你的程式可以存取它們。
如果這些鑰匙被別人拿到了，他們就能用你的名義做任何事——包括花你的錢。

### 你一定要做到的事

1. **永遠不要把密鑰直接寫在程式碼裡面**（這叫做「硬編碼」，Hard-coding）
   - ❌ 錯誤做法：`apiKey = "sk-12345abcde"`
   - ✅ 正確做法：把密鑰存在一個叫 `.env` 的特殊檔案裡，程式會自己去讀
2. **確認 `.env` 不會被上傳到 GitHub**
   - 有一個叫 `.gitignore` 的檔案，用來告訴 Git「哪些東西不要上傳」
   - `.env` 一定要被列在 `.gitignore` 裡面
3. **設定 Claude Code 的權限，不讓它讀取你的密鑰**
   - 在你的專案的 `.claude/settings.json` 裡面加入限制規則（CLAUDE.md 裡有詳細說明）

### 為什麼這麼重要？

> 真實案例：有人不小心把 AWS 的金鑰上傳到 GitHub，隔天收到一張幾萬美金的帳單。
> 因為有機器人專門在 GitHub 上掃描外洩的金鑰，一旦被找到就會被立刻盜用。

## 二、你不需要看懂每一行程式碼，但你需要理解它在做什麼

AI 產生的程式碼不一定正確。它有時候會「很自信地」寫出錯誤的東西。

### 實際的做法

- **請 AI 用白話解釋**：每次 AI 寫完一段程式，直接問它「用白話幫我解釋這段在做什麼」
- **問「如果拿掉這段會怎樣？」**：這能幫你理解每段程式的用途
- **請 AI 畫流程圖**：視覺化比讀程式碼容易理解很多
- **注意 AI 的「幻覺」**：AI 有時候會引用不存在的套件或 API，你可以問它「這個套件真的存在嗎？」

### 核心觀念

你不需要會修車，但你要知道車開起來有沒有問題。
同樣地，你不需要變成程式設計師，但你要有能力**驗證結果是否正確**。

## 三、單元測試（Unit Test）

### 什麼是單元測試？

就是寫一些小程式來「自動檢查」你的主程式有沒有正常運作。
就像你開車前會檢查油量、胎壓一樣，測試就是程式的健康檢查。

### 為什麼重要？

- AI 寫的程式看起來能跑，不代表它真的正確
- 有了測試，每次修改程式之後可以快速確認有沒有壞掉的地方
- 如果你以後要改功能或加新功能，測試是你的安全網

### 你可以做的

- 請 AI 在每次寫完功能後，順便寫對應的測試
- 跑測試看結果——綠色通過、紅色失敗，這是你最直觀的品質指標

---

## 四、版本控制（Git）

### 什麼是 Git？

Git 是一個「時間機器」，它可以記錄你的程式在每個時間點的狀態。
如果改壞了，你可以回到之前正常的版本。

### 你需要知道的

- **經常 commit**：每完成一個小功能就存檔一次，不要等做了很多才一次存
- **寫有意義的 commit message**：例如「新增登入功能」而不是「更新」
- **不要把不該上傳的東西上傳**：例如 `.env`、`node_modules`、build 出來的檔案
- **出事前先建新的分支**：讓 Claude Code 在新的分支上工作，確認沒問題再合併

---

## 五、錯誤處理（Error Handling）

### 為什麼重要？

如果程式出錯但沒有任何提示，你完全不知道問題在哪。
好的錯誤處理會告訴你「哪裡出了什麼問題」，讓你可以快速修復。

### 你可以做的

- 告訴 AI：「每個可能出錯的地方都要加上錯誤處理」
- 要求錯誤訊息要有意義——不是只顯示「Error」，而是「無法連線到資料庫：逾時」

---

## 六、程式碼品質與重構

### 什麼是重構？

就是在不改變功能的情況下，把程式碼整理得更乾淨、更好維護。
就像搬家之後重新整理衣櫃——東西還是一樣的，但擺得更有條理。

### 為什麼重要？

- AI 生成的程式碼經常是「能跑就好」的狀態
- 如果不整理，程式會越來越亂，最後變成沒人看得懂的「義大利麵條」
- 適時請 AI 幫忙重構，保持程式碼的可讀性

## 七、善用 CLAUDE.md 設定檔

Claude Code 會自動讀取專案根目錄的 `CLAUDE.md` 檔案，並按照裡面的指示來工作。

### 為什麼要用 CLAUDE.md？

- 讓 Claude Code 自動遵守你設定的規則（例如：永遠不要硬編碼密鑰）
- 自動建立安全的開發環境（`.gitignore`、`.env.example` 等）
- 確保每次開發都遵循最佳實踐

### 怎麼用？

你不需要自己一個一個去設定。在我們提供的 **CLAUDE.md** 裡，已經寫好了指令讓 Claude Code 自動幫你：

1. 建立安全設定檔 `.claude/settings.json`
2. 建立 `.gitignore` 避免上傳敏感資料
3. 建立 `.env.example` 作為環境變數模板
4. 驗證整體安全設定

你只需要把 CLAUDE.md 放到你的專案根目錄就好。

## 八、給 AI 好的指令

你對 AI 說的話（Prompt）越清楚，AI 的產出越好。

### 實用建議

- **把大任務拆成小步驟**：不要一次丟一大段需求，一步一步來效果更好
- **先讓 AI 做計畫**：請 Claude Code 先規劃再動手，例如說「先幫我列出要做哪些事情」
- **描述你要的結果**：「我需要一個使用者登入頁面，要有 email 和密碼欄位，登入後跳轉到首頁」
- **不要假設 AI 知道你的前文**：每次對話都把關鍵背景資訊帶上
